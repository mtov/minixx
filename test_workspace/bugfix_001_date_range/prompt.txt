Users reported that date-based exports are missing the last day of the selected range.

For example, when someone asks for data from `2024-01-10` to `2024-01-12`, they expect the result to include all three dates, not stop one day early.

Expected behavior:
- the range should include both the start date and the end date;
- dates should stay in `YYYY-MM-DD` format;
- if the end date is before the start date, the result should be empty.
