Fix `build_stages(tasks)` in `src/scheduler.py`.

The function receives a list of task dictionaries like
`{"name": "build", "deps": ["compile"]}` and must group them into execution stages.

Rules:
- each task must appear exactly once in the output;
- tasks in the same stage must have all dependencies satisfied by earlier stages;
- tasks that become ready at the same time must keep their original input order;
- raise `KeyError` if a task depends on a missing task;
- raise `ValueError` if there is a dependency cycle.

Do not modify the tests.
